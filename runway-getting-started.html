1Create your account & organization

An organization represents your integration — it owns API keys, credits, and config.

Do this

  • Sign up at dev.runwayml.com and create a new organization.
  • Apply your $5k dev-program credit to the org (or confirm it's already applied).
  • Invite teammates under Organizations & Roles if needed.
Heads up: API keys are org-scoped, not user-scoped. Removing a teammate does not revoke their key — you must disable keys manually.

2Generate an API key

  • Open the API Keys tab → create a key with a clear name (e.g. dev-local).
  • Copy it now — the secret is shown only once.
  • Store it in your shell / secret manager, never in code:
export RUNWAYML_API_SECRET="key_xxxxxxxxxxxxxxxx"
Best practice: separate keys per environment (staging/prod) and per developer. The SDKs read RUNWAYML_API_SECRET automatically.

3Confirm credits & autobilling

  • Verify the dev-program credits show on the org's Billing tab (1 credit = $0.01).
  • Optionally enable autobilling with a recharge threshold so jobs never stall on empty credits.
  • Make sure billing/suspension emails go to a monitored inbox (not spam).

4Install an SDK

pip install runwayml   # Python 3.8+
npm install @runwayml/sdk   # Node 18+, ships TypeScript types

B · The runway CLI (zero dependencies)

A stdlib-only Python script — nothing to pip install. It submits a task, polls every 5s, and prints or downloads the result.

export RUNWAYML_API_SECRET=key_xxx          # your dev-org key → spends the $5k

./runway_cli.py image "a neon koi pond at dusk" -o koi.png
./runway_cli.py video https://site.com/start.jpg --prompt "slow dolly in" -o clip.mp4
./runway_cli.py video ./start.jpg --model gen4_turbo --duration 5   # local file auto-encodes
./runway_cli.py task task_abc123            # check status / get URL
./runway_cli.py cancel task_abc123
./runway_cli.py org                          # tier, credits, org info

Defaults: image → gen4_image @ 1360:768; video → gen4.5 @ 1280:768, 5s. Override with --model / --ratio / --duration. Local images under 5MB are auto-encoded; host larger ones at an HTTPS URL. Covered by test_runway_cli.py (python3 -m unittest test_runway_cli).

Output URLs expire in 24h — always pass -o (or download promptly) for anything you want to keep.

5Make your first calls (SDK)

All generation is async: submit a task → poll → fetch the output URL. The SDK helper does the polling for you.

A · Text → Image (cheap, instant gratification)

from runwayml import RunwayML
client = RunwayML()  # reads RUNWAYML_API_SECRET

task = client.text_to_image.create(
    model="gen4_image",
    prompt_text="A beautiful sunset over a calm ocean",
    ratio="1360:768",
).wait_for_task_output()

print(task.output[0])  # URL to the generated image
import RunwayML from '@runwayml/sdk';
const client = new RunwayML();

const task = await client.textToImage.create({
  model: 'gen4_image',
  promptText: 'A beautiful sunset over a calm ocean',
  ratio: '1360:768',
}).waitForTaskOutput();

console.log(task.output[0]);

B · Image → Video

task = client.image_to_video.create(
    model="gen4.5",                 # or gen4_turbo (cheapest), veo3.1, seedance2
    prompt_image="https://example.com/start.jpg",
    prompt_text="slow cinematic dolly-in",
    ratio="1280:768",               # NOTE: use resolution strings, not 16:9
    duration=5,
).wait_for_task_output()

print(task.output[0])  # URL to the generated video
# 1) submit
curl -s https://api.dev.runwayml.com/v1/image_to_video \
  -H "Authorization: Bearer $RUNWAYML_API_SECRET" \
  -H "X-Runway-Version: 2024-11-06" \
  -H "Content-Type: application/json" \
  -d '{"model":"gen4_turbo","promptImage":"https://.../img.jpg","ratio":"1280:768","duration":5}'
# -> {"id":"task_abc"}

# 2) poll (interval >= 5s, add jitter + backoff)
curl -s https://api.dev.runwayml.com/v1/tasks/task_abc \
  -H "Authorization: Bearer $RUNWAYML_API_SECRET" \
  -H "X-Runway-Version: 2024-11-06"
# status: PENDING -> THROTTLED -> RUNNING -> SUCCEEDED | FAILED | CANCELED
Start & end frames: pass promptImage as an array of {uri, position} with position = "first" / "last" to pin the opening and closing frames.

6Know the task lifecycle

StatusMeaning
PENDINGAccepted, awaiting scheduling
THROTTLEDQueued — you hit your concurrency cap; runs in order automatically
RUNNINGGenerating
SUCCEEDEDDone — output URL(s) in the output/artifacts array
FAILEDError or moderation block — see failure / failureCode
CANCELEDCancelled via DELETE /v1/tasks/{id}
Poll at ≥ 5s with jitter. Retry 429/502/503/504 with exponential backoff — the SDKs do this for you.

7Pick the right model

Video

gen4_turbo · 5 cr/s — cheapest gen4.5 · 12 cr/s veo3.1 · 20–40 cr/s seedance2 · up to 4K aleph2 · video editing act_two · performance

Image

gen4_image_turbo · 2 cr gen4_image · 5–8 cr gpt_image_2 gemini_image3_pro gemini_2.5_flash · 5 cr

Test cheap, then scale. Prototype on gen4_turbo / gen4_image_turbo, switch to higher-quality models only once prompts are dialed in.

8Feeding in your own assets

MethodImage limitVideo limitNotes
HTTPS URL16 MB32 MBHTTPS + domain, valid Content-Type, no redirects
Base64 data URI5 MB16 MBdata:image/png;base64,… (+~33% size)
Ephemeral upload200 MBrunway:// URI, valid 24h, reusable

Images: JPEG / PNG / WebP (no GIF). Video: MP4, MOV, MKV, WebM, 3GP, OGV. The Content-Type header — not the file extension — decides validity.

9Pre-production checklist

  • API key in a secret manager; separate keys per env & per dev; run git grep "key_" to confirm nothing's committed.
  • Handle 429 / 502 / 503 / 504 with exponential backoff + jitter.
  • Handle FAILED tasks (read failure / failureCode); moderated jobs still cost full credits.
  • Validate inputs (size, codec, ratio, Content-Type) before submitting.
  • Autobilling on; billing/suspension emails monitored and un-spammed.
  • Confirm use case fits content policy; relax public-figure rules only via the contentModeration param when appropriate.
  • Monitor error rate, daily volume, credit burn, and throttling.

B · Portal walkthrough

  1. Sign in at dev.runwayml.com → open the Characters tab.
  2. Click Create a Character.
  3. Upload image: adjutant_source.png (or the upscaled 4K version).
  4. Voice: pick a preset, or design/clone one (spec below). Preview it.
  5. Instructions: paste system_prompt.txt.
  6. Knowledge base (optional): upload knowledge_base.txt and knowledge_base_agentic_disciplines.txt.
  7. Starting script (optional): paste the opening line below.
  8. Create Character → copy the Avatar ID (UUID) and save it.

C · Voice & opening line

Voice target: calm, measured, synthetic-feminine, mid-low register, confident — a composed mission-control operator with a faint digital texture. Design-prompt:

A calm, composed female AI voice. Measured and precise, mid-to-low
register, quietly confident, with a faint synthetic/processed texture
and a subtle dry edge. Unhurried, never bubbly.

Opening line (startScript):

General. Adjutant online. The fleet is standing by — what's our objective?

D · After creation — drive a session

  • Create a real-time session (model gwm1_avatars) → receive WebRTC credentials (retrievable once — recreate the session if the connection fails).
  • Session states: NOT_READY → READY → RUNNING → COMPLETED; max 5 minutes per session.
  • personality and startScript can be overridden per session for context injection.
  • Embed via the Runway widget / React integration or LiveKit Agents.
